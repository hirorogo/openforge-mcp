# テクスチャ編集

このガイドでは、OpenForge MCP のテクスチャ編集ツールを使って、アバターやワールドのテクスチャを加工する方法を解説します。髪色の変更、衣装のカラー変更、パターン生成、ロゴの合成など、画像編集ソフトを使わずに自然言語で実行できます。

## テクスチャ編集でできること

| ツール | 用途 | 例 |
|---|---|---|
| `adjust_hsv` | 色相・彩度・明度の調整 | 髪色を茶色から金色に変更 |
| `apply_gradient` | グラデーションの適用 | 衣装に上から下へ色のグラデーション |
| `generate_pattern` | パターンの生成 | ストライプ、チェック、ドットの生成 |
| `overlay_decal` | デカール（画像）の合成 | ロゴやマークを衣装に貼り付け |
| `swap_color` | 特定色の置換 | 赤い部分をすべて青に変更 |
| `batch_recolor` | 一括カラー変更 | 衣装全体の配色を変更 |
| `export_texture` | 加工済みテクスチャの書き出し | PNG / TGA で保存 |

## 髪色の変更 (adjust_hsv)

`adjust_hsv` は、テクスチャの色相 (Hue)、彩度 (Saturation)、明度 (Value) を調整するツールです。

### 基本的な髪色変更

**あなた:**
> 髪のテクスチャの色を変えて。今の茶色から金色にして

**AI の動き:** `adjust_hsv` が実行され、髪テクスチャの色相がシフトされます。

**AI の返答例:**

```
テクスチャ編集完了:
  対象: Hair_Texture.png
  色相シフト: +30 度
  彩度調整: +15%
  明度調整: +20%
  → 茶色から金色に変更しました
```

**あなた:**
> スクリーンショットで確認させて

### 細かい調整

**あなた:**
> もう少し赤みを足して。彩度も少し上げて

**AI の動き:** 色相と彩度が再調整されます。

**あなた:**
> 暗めの銀色にしたい。彩度を下げて、明度を上げて

**AI の動き:** 彩度が下げられ（色味を抑え）、明度が上げられます（明るくする）。結果として銀色に近い色合いになります。

:::tip 色変更のパラメータ
色相 (Hue) は 0 から 360 度の値で色の種類を指定します。彩度 (Saturation) は色の鮮やかさ、明度 (Value) は明るさです。自然言語で「赤っぽく」「暗めに」と指示すれば、AI が適切なパラメータを設定します。
:::

## グラデーションの適用 (apply_gradient)

衣装やオブジェクトにグラデーション（色の段階的変化）を適用します。

**あなた:**
> スカートのテクスチャに上から下へグラデーションをかけて。上が濃い紺色で、下が薄い水色に

**AI の動き:** `apply_gradient` が実行され、スカートのテクスチャに縦方向のグラデーションが適用されます。

**AI の返答例:**

```
グラデーション適用完了:
  対象: Skirt_Texture.png
  方向: 上 → 下
  開始色: #1a237e (濃い紺色)
  終了色: #81d4fa (薄い水色)
  ブレンドモード: 乗算
```

### グラデーションの方向

**あなた:**
> 靴下のテクスチャに左から右のグラデーションをかけて。白からピンクへ

**AI の動き:** 横方向のグラデーションが適用されます。

指定できる方向:

| 方向 | 説明 |
|---|---|
| 上 → 下 | 縦方向（最も一般的） |
| 下 → 上 | 逆方向の縦 |
| 左 → 右 | 横方向 |
| 右 → 左 | 逆方向の横 |
| 中心 → 外側 | 放射状 |

## パターンの生成 (generate_pattern)

ストライプ、チェック、ドットなどのパターンをテクスチャとして生成します。

### ストライプ

**あなた:**
> 赤と白のストライプパターンを生成して。縦縞で、幅は均等にして

**AI の動き:** `generate_pattern` が実行され、ストライプパターンのテクスチャが生成されます。

**AI の返答例:**

```
パターン生成完了:
  種類: ストライプ (縦)
  色 1: #e53935 (赤)
  色 2: #ffffff (白)
  ストライプ幅: 均等 (32px)
  解像度: 1024x1024
```

### チェック

**あなた:**
> 黒と白のチェック柄を生成して。各マスは 64px 四方で

**AI の動き:** チェックパターンが生成されます。

### ドット

**あなた:**
> 紺色の背景に白いドット柄を生成して。ドットの直径は 16px、間隔は 48px で

**AI の動き:** ドットパターンが生成されます。

### 生成したパターンの適用

**あなた:**
> 生成したストライプパターンを靴下のテクスチャに適用して

**AI の動き:** 生成されたパターンテクスチャが靴下のマテリアルに適用されます。

## デカールの合成 (overlay_decal)

既存のテクスチャの上に、別の画像（ロゴ、マーク、文字など）を重ねて合成します。

**あなた:**
> Tシャツのテクスチャに「logo.png」を胸の位置に合成して。サイズは横 200px で

**AI の動き:** `overlay_decal` が実行され、ロゴ画像がTシャツのテクスチャ上に合成されます。

**AI の返答例:**

```
デカール合成完了:
  ベーステクスチャ: TShirt_Texture.png
  デカール: logo.png
  位置: (412, 280) - 胸の中央付近
  サイズ: 200 x 150 px (アスペクト比維持)
  ブレンドモード: 通常
```

**あなた:**
> 位置を少し上に移動して。透明度を 80% にして

**AI の動き:** デカールの位置と透明度が調整されます。

:::info 合成位置の指定
テクスチャ上の位置はピクセル座標で厳密に指定することも、「胸の中央」「背中の上部」などの自然言語で指定することもできます。自然言語の場合、AI がテクスチャの UV マッピングを考慮して適切な位置を推定します。
:::

## 色の置換 (swap_color)

テクスチャ内の特定の色を別の色に置き換えます。

**あなた:**
> 衣装テクスチャの赤い部分をすべて青に変えて

**AI の動き:** `swap_color` が実行され、赤い色域のピクセルが青に置き換えられます。

**AI の返答例:**

```
色置換完了:
  対象: Outfit_Texture.png
  置換前: 赤色域 (H: 340-20, S: 50-100%, V: 30-100%)
  置換後: 青色域 (H: 220, S: 同値維持, V: 同値維持)
  置換ピクセル数: 48,320 / 1,048,576 (4.6%)
```

**あなた:**
> 緑の部分も黄色に変えて

**AI の動き:** 追加で緑色域が黄色に置換されます。

:::warning 色置換の範囲
色置換は色相の近い色を一括で変換します。境界部分のグラデーションが不自然になる場合は、色域の範囲を狭めるか、HSV 調整 (`adjust_hsv`) でのアプローチを検討してください。
:::

## 一括カラー変更 (batch_recolor)

衣装全体の配色をまとめて変更します。`swap_color` が単色の置換なのに対し、`batch_recolor` は複数の色を同時に変更して統一感のある配色に仕上げます。

**あなた:**
> 衣装の配色を全体的に変えて。今の赤系の配色を、青系に統一して

**AI の動き:** `batch_recolor` が実行され、テクスチャ全体の色相がシフトされます。

**AI の返答例:**

```
一括カラー変更完了:
  対象: Outfit_Texture.png
  変更内容:
    メインカラー: #c62828 (赤) → #1565c0 (青)
    サブカラー: #ef5350 (薄い赤) → #42a5f5 (薄い青)
    アクセントカラー: #b71c1c (暗い赤) → #0d47a1 (暗い青)
  配色の統一感を維持して変換しました
```

**あなた:**
> ボタンと縁取りの金色はそのまま残して

**AI の動き:** 金色域をマスクから除外して再実行されます。

## テクスチャの書き出し (export_texture)

加工したテクスチャをファイルとして書き出します。

**あなた:**
> 加工した髪のテクスチャを PNG で書き出して

**AI の動き:** `export_texture` が実行され、テクスチャファイルが保存されます。

**AI の返答例:**

```
テクスチャ書き出し完了:
  ファイル: Assets/Textures/Hair_Texture_edited.png
  解像度: 2048 x 2048
  形式: PNG (32bit RGBA)
  ファイルサイズ: 4.2 MB
```

:::tip テクスチャのバックアップ
テクスチャ編集は元のファイルを上書きせず、新しいファイルとして保存されます。元のテクスチャはそのまま残るので、いつでも元に戻せます。
:::

## 実践例: アバターの衣装カラーバリエーション

1 つの衣装テクスチャから複数のカラーバリエーションを作る手順です。

**あなた:**
> 衣装テクスチャのカラーバリエーションを 3 つ作って。赤系、青系、緑系で

**AI の動き:** 元のテクスチャを基に、3 つのカラーバリエーションが生成されます。

**AI の返答例:**

```
カラーバリエーション作成完了:
  1. Outfit_Red.png - 赤系 (元のテクスチャの色相を調整)
  2. Outfit_Blue.png - 青系 (色相シフト: +120 度)
  3. Outfit_Green.png - 緑系 (色相シフト: +240 度)
各テクスチャは Assets/Textures/ に保存されました
```

**あなた:**
> 3 つのバリエーションを並べたスクリーンショットを見せて

ここで各バリエーションの仕上がりを確認します。

## テクスチャ編集のベストプラクティス

| 項目 | 推奨 |
|---|---|
| 解像度 | アバター: 2048x2048 以下、ワールド: 用途に応じて 512-2048 |
| ファイル形式 | PNG（透過あり）、TGA（Unity 標準） |
| 色空間 | sRGB（テクスチャ）、Linear（ノーマルマップ） |
| ファイルサイズ | VRAM 消費を抑えるため、不要に大きくしない |

:::warning テクスチャサイズとパフォーマンス
テクスチャサイズが大きいほど VRAM を消費します。VRChat のパフォーマンスランクにも影響するため、必要最小限のサイズに抑えることを推奨します。4096x4096 のテクスチャは、目に見える改善がない場合は 2048x2048 に縮小してください。
:::
