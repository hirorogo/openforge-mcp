# 便利な使い方とコツ

OpenForge MCP をより効果的に使うためのテクニック集です。


## スクリーンショットで結果を確認する

AIに操作結果を見せることで、視覚的なフィードバックを得られます。

```
スクショ撮って確認して
```

AIがスクリーンショットを撮影し、現在の状態を分析してくれます。「思ったのと違う」場合は、そのまま修正を依頼できます。

```
スクショ撮って。もう少し明るい感じにして
```


## アプリ間連携

Blender と Unity を横断した操作が一言で指示できます。

```
Blenderで作ったモデルをFBXでエクスポートしてUnityにインポートして
```

```
BlenderでCubeをモデリングして、UnityのAssetsフォルダに持っていって
```

AI が自動的に複数アプリの操作を順番に実行します。


## 一括操作

複数のオブジェクトに同じ操作を適用したいときは、まとめて指示できます。

```
全部のオブジェクトにRigidbodyをつけて
```

```
シーン内のライトを全部2倍の明るさにして
```

```
名前にEnemyが含まれるオブジェクトを全部削除して
```


## レシピ機能

レシピは、よく使う一連の操作をまとめたものです。

### レシピ一覧を確認する

```
レシピ一覧を見せて
```

### レシピを実行する

```
VRChatワールドの基本セットアップレシピを実行して
```

```
FPS用プレイヤーのレシピを実行して
```

レシピは複数のツールを順番に呼び出し、複雑なセットアップを自動化します。


## 安全な操作のために

### 変更前にセーブする

大きな変更をする前は、AIにセーブを頼んでおくと安心です。

```
セーブしてからマテリアルを全部変えて
```

```
保存してから、オブジェクトを再配置して
```

### 実験用のブランチを使う

バージョン管理と組み合わせて、実験的な変更を安全に試せます。

```
実験ブランチを作って、ライティングを全部変えてみて
```

うまくいかなければ元のブランチに戻るだけで済みます。

```
元のブランチに戻して
```


## パフォーマンス分析

プロジェクトのパフォーマンスを AI に分析させることができます。

```
パフォーマンスを分析して
```

```
シーンのポリゴン数とマテリアル数を教えて
```

```
重いオブジェクトを探して
```

VRChat アバターの場合は、パフォーマンスランクの確認と改善提案も得られます。

```
パフォーマンスランクを確認して、改善点を教えて
```


## AIテストプレイ

AI にプロジェクトをテストさせることができます。

```
テストプレイしてバグを探して
```

```
ゲームを実行して、UIが正しく表示されるか確認して
```

```
全シーンを順番に開いて、エラーがないか確認して
```

AI はゲームを実行し、スクリーンショットやログを分析して問題点を報告します。


## 複合的な使い方の例

### モデリングからインポートまで

```
Blenderで椅子のモデルを作って、UVを展開して、FBXでUnityに持っていって、
シーンの (3, 0, 0) に配置して
```

### VRChat アバターの仕上げ

```
パフォーマンスランクを確認して、Poorなら自動最適化して、
もう一回ランクを確認して
```

### シーンの一括調整

```
全部のPointLightの色を暖色系にして、強さを半分にして、
スクショを撮って見せて
```
